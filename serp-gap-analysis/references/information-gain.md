# Phase 4: Information Gain

## Purpose

A useful content brief should not just mirror the SERP. It should incorporate what the user genuinely knows or offers that competitors lack, then express that differentiation in the outline.

## Collecting context

Use the user's reply to the Phase 2 closing question as the default source of business or product context.

## What counts as information gain

Information gain is the unique value the user can add beyond what already exists in the SERP. In practice this comes from:

- **A product they sell**: features, pricing, integrations, use cases competitors can't demonstrate
- **A feature they built**: technical implementation details, design decisions, performance data
- **Real customer experience**: case studies, support patterns, common questions, success stories
- **Internal workflows**: implementation knowledge, operational processes, team practices
- **Original data**: benchmarks, surveys, experiments, analytics, A/B test results
- **Firsthand expertise**: years of domain experience, certifications, unique perspective

## Applying information gain to the outline

1. Start with the table-stakes topics from Phase 2 — these must be covered for the page to be competitive
2. Identify which topics the user's context can enhance or go deeper on than competitors
3. Add sections that are only possible because of the user's unique context
4. For each H2 in the outline, note whether it's:
   - **Table stakes** — must cover, but competitors already do this well
   - **Improvement opportunity** — competitors cover it, but the user can do it better with their context
   - **Unique section** — only this user can write this because of their information gain

## Ground rules

- Do not fabricate EEAT or differentiation when the source context does not support it
- If the user's context is thin, be honest about it — a brief with 1-2 genuine angles is better than 5 invented ones
- The outline should weave table-stakes topics with information gain naturally, not separate them into "generic" and "unique" buckets
