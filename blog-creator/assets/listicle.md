# Listicle

> When to use: "Top N...", "Best...", curated resource lists, tool roundups, strategy collections
> Minimum length: 1,200 words

## Bones (required)

1. **Selection criteria** — how and why these items were chosen. Brief but transparent.
2. **The list** — each item gets a section. The depth per item should match the format: product roundups need features/pricing, strategy lists need explanations/examples. Adapt the structure to what's being listed.

## Moves (pick 1-3 as relevant)

- **Quick comparison table** — scannable overview of all items, especially useful for product/tool lists
- **How to choose** — situational guidance ("If you need X, go with item Y")
- **Pros and cons** — per item, but only when the items are products or tools with genuine trade-offs
- **FAQ** — common questions about the category
- **TL;DR** — top pick + runner-up with brief reasoning

## Guidance

- The list order should be intentional. Best-first, cheapest-first, simplest-first — pick a logic and state it.
- Item sections don't all need the same structure. A standout item might deserve 300 words. A niche pick might need 80. Match depth to how much there is to say.
- "Best for [specific use case]" labels in headings help scanners find what they need fast.

## Anti-patterns

- Don't force Pros/Cons/Pricing on every item when the list isn't about products (e.g., "10 strategies for X" doesn't have pricing).
- Don't include filler items to hit a round number. 7 genuinely useful items beats 10 padded ones.
- Don't describe every item with the same sentence structure. Vary the rhythm.
