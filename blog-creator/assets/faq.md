# FAQ / Q&A

> When to use: Common questions pages, knowledge base articles, "People Also Ask" targeting
> Minimum length: 1,200 words

## Bones (required)

1. **Questions in natural language** — phrased the way people actually search, not how a textbook would title a section.
2. **Direct answers first, then depth** — each answer opens with a clear, concise response before elaborating. The first sentence should work as a standalone answer.

## Moves (pick 1-3 as relevant)

- **Quick answers table** — one-sentence answer per question at the top for scanners
- **Grouping by category** — organize questions into themes (basics, getting started, advanced, troubleshooting) when there are 8+ questions
- **Comparison table** — when a question involves "X vs Y" distinctions
- **Visual explanations** — diagrams or charts for complex answers
- **Related resources** — links to deeper content for readers who want more

## Guidance

- The number of questions should match the topic. Some topics have 5 real questions. Others have 15. Don't force a count.
- Answer depth should vary. Some questions need 50 words. Others need 200 with an example. Match the answer to the question's complexity.
- Progress from fundamental to advanced questions. The reader who needs basic answers shouldn't have to scroll past expert-level ones.

## Anti-patterns

- Don't write questions you want to answer — write questions people actually ask. Check "People Also Ask" and search suggestions.
- Don't give every answer the same length. A yes/no question doesn't need 200 words.
- Don't use FAQ format as a dumping ground for content that should be a proper guide. If 5 questions are all about the same subtopic, that's an article, not an FAQ.
